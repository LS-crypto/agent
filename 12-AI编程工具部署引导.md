## 二、用户偏好（AI 必须遵守）

1. **不要建议新开第二台 ECS**（试用额度限制，用户已确认新开要多花钱）
2. **优先 Docker 部署**，ECS 上只装 Docker，不要在 ECS 上装 JDK/Maven（除非用户明确要求 jar 方式）
3. **Ubuntu 用 Workbench + root**，不要再用 `Administrator` + RDP（除非系统改回 Windows）
4. **本机构建、ECS 运行**：`mvn package` 和 `docker build` 在用户 Windows 本机；ECS 只做 `docker load/run`
5. **镜像传输**：优先 `docker save` → Workbench 上传 tar → `docker load`（ECS 拉 Docker Hub 很慢，`hello-world` 失败可忽略）
6. **Ubuntu 装 Docker 只用**：`apt-get install -y docker.io`（**不要**写 `docker-compose-plugin`，默认源没有会导致安装失败）
7. **容器启动参数必须带**：`--restart unless-stopped`
8. **文档与注释**：用户接受中文；部署说明写入 `云服务器部署/` 或项目 README，不要过度创建无关 md
9. **PowerShell**：本机命令用 PowerShell 语法；ECS 用 bash
10. **不要自动 git commit / 不要新开 ECS / 不要改阿里云密码**

## 三、标准项目结构（AI 生成新项目时应包含）

```
<项目名>/
├── pom.xml                          # Java/Spring Boot 项目
├── Dockerfile                       # 必须：基于 eclipse-temurin JRE，COPY jar
├── docker-compose.yml               # 可选：restart: unless-stopped
├── src/...
├── deploy-package/                  # 可选：jar 备用部署
│   └── （jar 构建后放入）
└── README.md                        # 含本地运行 + Docker 部署说明
```

### Dockerfile 模板（与测试系统一致）

```dockerfile
FROM eclipse-temurin:17-jre-alpine
WORKDIR /app
COPY target/<artifactId>-<version>.jar app.jar
EXPOSE 8080
ENTRYPOINT ["java", "-jar", "app.jar", "--spring.profiles.active=prod"]
```

## 四、AI 执行部署的标准流程

当用户说「部署到服务器」「上线」「发布到 ECS」时，AI 按以下顺序：

### 阶段 A — 本机（Windows）

```powershell
cd D:\system\docker问题排查\<项目名>
mvn clean package -DskipTests
docker build -t <项目名>:<版本> .
docker run -d -p 8080:8080 --name <项目名>-local <项目名>:<版本>
# 验证 http://localhost:8080 后删除本地容器
docker stop <项目名>-local; docker rm <项目名>-local
docker save -o <项目名>.tar <项目名>:<版本>
```

### 阶段 B — 告知用户上传

提示用户：Workbench **上传** `<项目名>.tar` 到 ECS `/opt/<项目名>/`

（AI 无法替用户上传文件到 ECS，必须明确提示这一步）

### 阶段 C — ECS（bash，root）

```bash
mkdir -p /opt/<项目名>
cd /opt/<项目名>

# 首次需装 Docker（若未装）
apt-get update && apt-get install -y docker.io
systemctl start docker && systemctl enable docker

# 部署/更新
docker stop <项目名> 2>/dev/null; docker rm <项目名> 2>/dev/null
docker load -i <项目名>.tar
docker run -d -p <端口>:8080 --name <项目名> --restart unless-stopped <项目名>:<版本>

# 验证
docker ps
curl http://localhost:<端口>/api/hello
```

### 阶段 D — 公网验证

```
http://118.178.144.109:<端口>
```

若外网不通：检查安全组是否放行 `<端口>`，容器是否 Up（`docker ps`）。

## 五、多项目规则（同一台 ECS）

| 项目 | 容器名 | 宿主机端口 | 安全组 |
|------|--------|------------|--------|
| 项目 A | project-a | 8080 | 8080/8080 |
| 项目 B | project-b | 8081 | 8081/8081 |
| 项目 C | project-c | 8082 | 8082/8082 |

更新某项目前：`docker stop` + `docker rm` 该容器，再 `load` + `run`。

## 六、AI 应主动生成的文件（用户要求部署时）

1. `Dockerfile`（若没有）
2. `docker-compose.yml`（可选，ports + restart）
3. 项目 `README.md` 中的 **部署章节**（本机 build + ECS run）
4. 如需脚本：
   - 本机：`scripts/docker-build.ps1`
   - ECS：`scripts/docker-deploy.sh`

不要生成依赖 `docker-compose-plugin` 的命令；Ubuntu 上 compose 可选：`apt install docker-compose` 或纯 `docker run`。

## 七、常见问题（AI 直接查，勿重复踩坑）

| 现象 | AI 应给出的答案 |
|------|----------------|
| 外网超时，localhost 正常 | 安全组缺端口，不是程序问题 |
| `docker build` 报错缺参数 | 应为 `docker build -t name:tag .` |
| `docker save` 报错 | 应为 `docker save -o file.tar image:tag` |
| ECS `docker-compose-plugin` 找不到 | 只装 `docker.io` |
| 用户说 localhost 看到别的项目 | 8080 被占，先 `docker ps` / `docker stop` |
| 换 Workbench 页面要不要重装 | 不用，Docker 装在 ECS 上 |
| 要不要新开 ECS | 不要，一台多端口 |
| hello-world 拉不下来 | 跳过，用 docker load |
| Windows 远程桌面 | 当前系统是 Ubuntu，用 Workbench root |
