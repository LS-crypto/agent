## 2. 用户文件存在哪里？

**已在代码里设计好路径**（ECS 上需在 Docker 挂载 volume 才持久）：

```text
runtime/workspaces/{用户ID}/projects/   ← Agent 创建/读写的代码与文件
runtime/db/sessions.sqlite              ← 对话记录（按 user_id 列）
runtime/logs/日期/{用户ID}.jsonl       ← 工具调用、Loop 轮次（主管审计）
```

每个用户 **只能** 访问自己的 `projects/`，工具层有路径校验。

**生产必做（阶段 L）：**

```bash
docker run -v /opt/sheldon-agent/runtime:/app/runtime ...
```

否则重建容器可能丢用户文件。

## 3. 主管怎么看到用户行为？

**已有基础：** `ActivityLogger` 把 `user_message`、`tool_call`、`tool_result` 等写入 JSONL；`view_activity.py` 可命令行查看。

**要做到你说的「售后 + 全知道」→ 阶段 K：**

- **Admin Web** `/admin`：用户列表、会话、工具时间线、拉黑按钮  
- **Admin CLI**：`sheldon-admin user show <id>`、`stats`、`ban`  
- **仪表盘**：注册人数、今日活跃、当前并发  

**隐私合规：** 需在用户协议中说明：为运维与售后，主管可查看对话与工具记录；Key 仅加密存储、不明文展示。

## 4. 安全与 BYOK

| 措施 | 说明 |
|------|------|
| Key 加密存库 | AES + 服务器 `MASTER_SECRET` |
| 传输 | HTTPS（阶段 J4） |
| 日志脱敏 | 扩展 `sanitize.py`，禁止 Key 进 JSONL |
| 鉴权 | 所有 API 带 JWT，禁止伪造 user_id |
| 拉黑 | `status=banned` 立即失效 |
| 限流 | 每用户 RPM + 全站最大并发 |
| 8081 | 仅本机，公网只 80 + 登录 |

普通用户：**必须** 填自己的 DashScope Key 才能聊；**你的** Key 只给 admin 或演示兜底。

---

## 5. 多少人能用？怎么拉黑？

- **注册上限：** `MAX_REGISTERED_USERS` 或邀请码（J3）  
- **同时在线/chat 上限：** 并发连接数或队列（J3）  
- **拉黑：** 后台一键 ban，写审计记录（J3 + K1）  
- **统计：** Admin 仪表盘 DAU / 总用户（K0）

---

## 6. CLI 怎么给用户？

**推荐路径（由易到难）：**

1. **大多数用户** → 只用 **Web**（已部署），登录即可。  
2. **进阶用户** → 安装 CLI，配置 **你的 API 地址 + 登录 token**，对话走云端，Key 仍在 Web 设置里（阶段 M2）。  
3. **开发者** → `uv tool install` 或 GitHub Releases 二进制（M3/M4）。

**不要** 把平台 DashScope Key 打进 CLI 安装包。

## 7. 建议开发顺序

```text
J0 登录隔离  →  J1 BYOK  →  J2 主管模型  →  J3 拉黑/限流
     →  K1 主管后台  →  L volume+配额  →  M CLI 连云端  →  J4 HTTPS
```

公网已开放，**J0 + J1 + J3（限流）** 建议优先，避免陌生人刷你的 Key。
