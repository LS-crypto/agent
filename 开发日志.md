### 2026-07-07 · N1c 三栏列宽拖拽（useColumnResize）

- **背景：** N1c Cursor 式三栏布局需要用户可自由拖拽调整侧栏 / Agent 面板宽度，初版 pointermove 每次 setState 导致全 App re-render 卡顿。
- **完成（新增）：**
  - `apps/web/src/useColumnResize.ts`（195 行）：列宽管理 hook
    - 拖拽中直接 `document.documentElement.style.setProperty()` 更新 CSS 变量 `--sidebar-w` / `--agent-w`，**零 re-render**
    - 松手（pointerup）时才 `setState` + 写入 `localStorage`（`sheldon_sidebar_w` / `sheldon_agent_w`）
    - `ResizeObserver` 监听容器宽度变化，自动 clamp 到 `[min, 45%容器宽]`
  - `App.tsx`：集成 hook + 渲染 2 个 `.split-handle-col` 垂直手柄（`.split-handle-grip-v` 握把）
  - `App.css`：手柄样式 + 768px 以下 `display:none`（左右栏变 overlay 抽屉模式）
- **完成（CSS 冲突修复）：**
  - 删除 1440px media query 中硬编码 `--sidebar-w: 300px` / `--agent-w: 340px`（hook 已接管）
  - 删除 769-900px media query 中 `.sidebar { width: 220px }`（覆盖 CSS 变量导致无法自由调整）
- **架构关键点：**
  ```css
  :root { --sidebar-w: 280px; --agent-w: 340px; }  /* hook 通过 inline style 动态覆盖 */
  .sidebar { width: var(--sidebar-w); }
  .agent-panel.open { width: var(--agent-w); }
  ```
  拖拽流程：`pointerdown → pointermove(DOM 更新 CSS 变量) → pointerup(setState + localStorage)`
- **涉及文件：** `useColumnResize.ts`（新建）、`App.tsx`、`App.css`
- **验收：** `npm run build` 通过 — 70 modules, 72.77 kB CSS, 277.52 kB JS, 无错误
- **对应任务清单：** N1c 三栏布局（部分）· 响应式 768px 手柄隐藏

### 2026-07-07 · 云端工作区 zip 上传（ECS 替代「打开本机文件夹」）

- **背景：** ECS 上用户无法使用「打开本机文件夹」（远程 Docker 访问不到用户电脑路径；`ALLOW_LOCAL_FOLDER` 公网默认关）。确定产品方案：**本地/桌面 = 打开文件夹 · 公网 Web = 上传 zip 到云端沙箱**。
- **完成（后端）：**
  - `server/services/workspace_upload.py`：zip 解析、zip slip 防护、`merge`/`subdir`/`replace` 三模式、`strip_root` 单层目录展开、敏感文件跳过（`is_sensitive_path`）、配额预检（`USER_WORKSPACE_QUOTA_BYTES`）
  - `POST /api/workspace/upload`（`multipart/form-data`）· `WorkspaceUploadResponse` schema
  - 若用户曾绑本机目录（`mode=local`），上传后自动 `reset_to_sandbox` 并返回 `switched_to_sandbox: true`
  - 依赖：`python-multipart`（`pyproject.toml` / `uv.lock`）
- **完成（前端）：**
  - `UploadWorkspaceModal.tsx` + CSS：拖拽/选择 zip、解压方式、进度条、配额提示；ECS 文案说明「请上传 zip」
  - `WorkspacePanel` 顶栏 **「上传」** 按钮；`App.tsx` 上传成功 toast + 刷新文件树 + `root_entry` 高亮
  - `api/client.ts`：`uploadWorkspace()`（XHR + 上传进度）
- **部署配套：** `deploy/nginx-sheldon.conf` · `client_max_body_size` 20m → **55m**（对齐 50MB zip 上限）；`.env.example` 补充 `WORKSPACE_UPLOAD_*` / 配额说明
- **涉及文件：** `workspace_upload.py`、`server/routes/workspace.py`、`server/schemas.py`、`UploadWorkspaceModal.*`、`WorkspacePanel.tsx`、`App.tsx`、`client.ts`、`nginx-sheldon.conf`、`.env.example`、`tests/test_workspace_upload.py`
- **验收：** `uv run pytest tests/test_workspace_upload.py -q` → 10 passed；`cd apps/web && npm run build` 通过
- **待办：** 本地手动走查 upload → **ECS 重部署** → 后续 **APK 打包**（`apps/web/android` Capacitor）

### 2026-07-07 · 工作区问题梳理 · 任务清单对照 · 迭代路线

- **ECS「打不开文件夹」结论：** 非部署 bug，是架构限制（浏览器填的本机路径在 Linux 容器内不存在）。ECS 正确能力为每人独立云端沙箱 `runtime/workspaces/{user_id}/projects/`。
- **任务清单对照（代码 vs 文档）：** 阶段 G、J0–J3 多用户、K1 Admin、N1 文件树、N3 打开文件夹等已在代码中实现，但 `任务清单.md` / 本日志部分条目未同步勾选；真正缺口为 N0 桌面 EXE、视觉 Web 发图、delegate 接入、HTTPS/J4、OSS 备份、远程 CLI 等。
- **视觉模型：** 目录已含 `qwen3-vl-plus` 等（`supports_vision`），但 Web/API/Loop 尚无图片上传与多模态消息，catalog 注释「图片输入待 Web 上传能力」— 列入后续开发。
- **约定迭代顺序：**
  1. 工作区（upload + 本地打开文件夹）← 当前
  2. 后续功能（视觉发图、delegate 等）
  3. ECS 重部署 + APK 打包
