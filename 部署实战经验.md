# 阿里云 ECS + Docker + 手机 APK 部署实战经验

> 基于 **Sheldon Agent** 项目（2026-07）一次完整上线总结。  
> 适用于：本机 Windows 开发 → 阿里云 ECS 公网部署 → Capacitor Android APK 内测。  
> 案例服务器：`118.178.144.109` · 部署目录：`/opt/sheldon-agent/`

---

## 一、架构一览

```text
本机 Windows                              阿里云 ECS
─────────────                             ────────────
deploy-build.ps1  →  sheldon-agent.tar  →  docker load + ecs-run.sh（后端 :8081）
deploy-web.ps1    →  web-dist.tar      →  setup-nginx.sh（网页 :80，/api 反代）
build-apk.ps1     →  sheldon-agent-debug.apk  →  发给测试者（不上传 ECS）
ecs-run.env       →  单独上传            →  密钥注入 docker run -e
```

| 端口 | 用途 |
|------|------|
| **80** | Nginx：静态前端 + `/api` 反代到 8081 |
| **8081** | Docker 后端 API（建议安全组仅内网或按需开放） |

**原则：密钥不进 Git、不进镜像**，通过 `ecs-run.env` + `docker run -e` 注入。

## 二、目录与文件清单

### 本机（Windows）

| 路径 | 说明 |
|------|------|
| `sheldon-agent/.env` | 本地开发密钥（git 忽略） |
| `sheldon-agent/deploy/ecs-run.env.example` | ECS 环境变量模板（进 Git） |
| `sheldon-agent/deploy/ecs-run.env` | 真实密钥（**勿提交 Git**） |
| `sheldon-agent/sheldon-agent.tar` | 本机 docker save 产物（约 80 MB） |
| `sheldon-agent/deploy/web-dist.tar` | 前端静态包（约 1 MB） |
| `sheldon-agent/deploy/sheldon-agent-debug.apk` | APK 产物（约 6 MB） |

### ECS（Ubuntu）

| 路径 | 说明 |
|------|------|
| `/opt/sheldon-agent/` | 统一部署目录 |
| `/opt/sheldon-agent/runtime/` | Docker volume，用户数据/SQLite/日志（**重建容器不删**） |
| `/opt/sheldon-agent/ecs-run.env` | 服务端私密配置 |
| `/opt/sheldon-agent/web/` | Nginx 静态根目录 |

## 三、首次部署（从零到公网可访问）

### 3.1 准备 `ecs-run.env`

```powershell
cd D:\system\Sheldon-Shuo-Agent\sheldon-agent\deploy
copy ecs-run.env.example ecs-run.env
notepad ecs-run.env
```

从本机 `.env` 对齐字段（示例）：

| 变量 | 必填 | 说明 |
|------|------|------|
| `DASHSCOPE_API_KEY` | ✅ | 百炼 API Key |
| `JWT_SECRET` | ✅ | `openssl rand -hex 32` |
| `MASTER_SECRET` | ✅ | 另一个随机 hex |
| `ADMIN_EMAIL` / `ADMIN_PASSWORD` | 建议 | 启动时创建管理员 |
| `CORS_ORIGINS` | 建议 | 含公网 IP + `https://localhost`（APK 需要） |
| `GITHUB_TOKEN` | 可选 | GitHub MCP |

**不要**把 `HOST`/`PORT`/`ALLOW_LOCAL_FOLDER` 写进 ECS env（由脚本或 Docker 处理）。

本机保存时用 **LF 换行**（见下文「坑 1」）。

### 3.2 本机构建产物

```powershell
cd D:\system\Sheldon-Shuo-Agent\sheldon-agent

# 后端镜像（需 Docker Desktop 已启动）
.\scripts\deploy-build.ps1

# 前端静态包
.\scripts\deploy-web.ps1
```

### 3.3 上传到 ECS `/opt/sheldon-agent/`

Workbench 上传（或 scp）：

- `sheldon-agent.tar`
- `deploy/web-dist.tar`
- `deploy/ecs-run.env`
- `deploy/ecs-run.sh`
- `deploy/setup-nginx.sh`
- `deploy/nginx-sheldon.conf`

### 3.4 ECS 上执行

```bash
cd /opt/sheldon-agent

# 去掉 Windows 换行（必做）
sed -i 's/\r$//' ecs-run.env ecs-run.sh setup-nginx.sh
chmod +x ecs-run.sh setup-nginx.sh

# 加载密钥
set -a && source ecs-run.env && set +a

# 后端
./ecs-run.sh

# 前端 + Nginx
./setup-nginx.sh

# 验收
curl -s http://127.0.0.1:8081/health
curl -s http://127.0.0.1/health
```

浏览器访问：`http://118.178.144.109/`

### 3.5 安全组

| 端口 | 是否放行 | 说明 |
|------|----------|------|
| TCP 80 | ✅ | 网页 + APK 的 `/api` |
| TCP 8081 | 按需 | 直连 API 调试；生产可仅内网 |

## 四、升级部署（项目更新后覆盖）

> **推荐路径：本机 build → 上传 tar → ECS load**，避免在 ECS 上拉 Docker Hub。

### 4.1 本机

```powershell
cd D:\system\Sheldon-Shuo-Agent\sheldon-agent
.\scripts\deploy-build.ps1    # 新 sheldon-agent.tar
.\scripts\deploy-web.ps1      # 新 web-dist.tar
```

### 4.2 上传并覆盖

只需上传变更文件（`ecs-run.env` 密钥未变可不传）。

### 4.3 ECS

```bash
cd /opt/sheldon-agent
sed -i 's/\r$//' ecs-run.env ecs-run.sh setup-nginx.sh 2>/dev/null || true
set -a && source ecs-run.env && set +a

./ecs-run.sh          # load 新镜像 + 重启容器（runtime 保留）
./setup-nginx.sh      # 更新静态页

curl -s http://127.0.0.1:8081/health
```

**不要用** `ecs-build-and-run.sh` 做常规升级——它会在 ECS 上 `docker build`，国内常因 Docker Hub 超时失败。

## 五、Android APK 打包（本机）

APK **不上传 ECS**，在本机构建后发给测试者。

### 5.1 前置

- Node.js + `apps/web` 已 `npm install`
- Android SDK（`ANDROID_HOME` 或 `D:\Android\SDK`）
- `apps/web/.env.android` 指向 ECS API，例如：
  ```env
  VITE_API_BASE=http://118.178.144.109/api
  VITE_PLATFORM=android
  ```

### 5.2 一键打包

```powershell
cd D:\system\Sheldon-Shuo-Agent\sheldon-agent
.\scripts\build-apk.ps1
```

产物：`deploy\sheldon-agent-debug.apk`

### 5.3 测试者安装

1. APK 传到 Android 手机  
2. 允许「未知来源」安装  
3. 打开 App → 注册/登录（连 ECS 公网 API）

## 六、踩坑与解决方案

### 坑 1：Windows 换行符 CRLF

**现象：**

```text
/usr/bin/env: 'bash\r': No such file or directory
ecs-run.env: line 3: $'\r': command not found
```

**解决（ECS）：**

```bash
sed -i 's/\r$//' ecs-run.env ecs-run.sh setup-nginx.sh ecs-build-and-run.sh
```

**预防（本机）：** 保存 `ecs-run.env` 时用 LF；或在 VS Code 右下角选 `LF`。

---

### 坑 2：ECS 上 Docker Hub 超时

**现象：**

```text
failed to resolve reference "docker.io/library/python:3.11-slim": i/o timeout
```

**方案 A（推荐）：** 本机 `deploy-build.ps1` → 上传 `sheldon-agent.tar` → `./ecs-run.sh`

**方案 B：** ECS 配置阿里云镜像加速器后重试 build：

```bash
# /etc/docker/daemon.json
{
  "registry-mirrors": ["https://你的ID.mirror.aliyuncs.com"]
}
sudo systemctl daemon-reload && sudo systemctl restart docker
```

---

### 坑 3：本机 Docker 未启动

**现象：**

```text
failed to connect to the docker API at npipe:////./pipe/dockerDesktopLinuxEngine
```

**解决：** 打开 Docker Desktop，等状态为 Running，再跑 `deploy-build.ps1`。

---

### 坑 4：PowerShell 脚本 `($mb MB)` 解析错误

**现象：**

```text
表达式或语句中包含意外的标记"MB"
```

**原因：** PowerShell 把 `MB` 当数字单位。

**解决：** 用格式化字符串，避免双引号内 `($mb MB)`：

```powershell
Write-Host ('[OK] {0} ({1} megabytes)' -f $path, $sizeMb) -ForegroundColor Green
```

---

### 坑 5：Workbench 上传会话过期

**现象：** `Session has expired, please login again`

**解决：** 重新登录 Workbench；或 `nano ecs-run.env` 粘贴内容。

---

### 坑 6：只重启容器 vs 重新 build

| 场景 | 做法 |
|------|------|
| 只更新 `ecs-run.env` 密钥/CORS | `source ecs-run.env` + `docker stop/rm` + `docker run` 或 `./ecs-run.sh` |
| 代码升级 | 必须新 `sheldon-agent.tar` + `./ecs-run.sh` |
| 只改前端 | 上传 `web-dist.tar` + `./setup-nginx.sh` |

`runtime/` 卷挂载不变，**用户数据和 SQLite 不会丢**。

## 八、迁移到其他项目的改造点

复制本流程到新项目时，替换以下占位即可：

| 占位 | Sheldon Agent 示例 | 你的新项目 |
|------|-------------------|------------|
| 部署目录 | `/opt/sheldon-agent/` | `/opt/你的项目/` |
| 镜像名 | `sheldon-agent:1.0.0` | `你的项目:版本` |
| 容器端口 | 8081 → 8765 | 宿主机端口 → 容器 PORT |
| 环境变量文件 | `ecs-run.env` | 同名或 `.env.production` |
| 启动脚本 | `ecs-run.sh` | 从 `docker run` 参数提取 |
| 前端反代 | `nginx-sheldon.conf` | 改 `root` 和 `proxy_pass` |
| APK API 地址 | `VITE_API_BASE=http://IP/api` | 改成你的公网地址 |

**通用脚本模板建议：**

```text
scripts/
  deploy-build.ps1    # 本机 docker build + save
  deploy-web.ps1      # 前端 build + tar
  build-apk.ps1       # （若有移动端）Capacitor + gradlew
deploy/
  Dockerfile
  ecs-run.env.example
  ecs-run.sh          # load + docker run
  setup-nginx.sh
  nginx-*.conf
```
